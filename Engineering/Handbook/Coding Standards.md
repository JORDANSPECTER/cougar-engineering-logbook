# Coding Standards

---

# Purpose

Code should remain readable, maintainable, and scalable.

Future engineers should understand the code without requiring extensive explanation.

---

# Standards

## Readability

Prefer readable code over clever code.

---

## Naming

Use descriptive names.

Avoid abbreviations unless universally understood.

---

## Functions

Functions should perform one responsibility.

Keep functions concise whenever practical.

---

## Components

Components should remain reusable.

Avoid tightly coupling components to individual pages.

---

## Comments

Explain why.

Do not comment what the code obviously does.

---

## Consistency

Follow established project patterns before introducing new ones.

---

## Refactoring

Leave code cleaner than you found it whenever practical.

---

## Testing

Engineering work should be validated before being considered complete.

---

Good code is written once.

Great code remains understandable years later.
