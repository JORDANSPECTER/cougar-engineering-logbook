# Definition of Done

---

# Purpose

Engineering work is considered complete only when all required engineering activities have been finished.

Writing code alone does not complete a mission.

---

# Definition of Done Checklist

Before closing an engineering mission:

## Engineering

- Feature implemented
- Code reviewed
- Errors resolved
- Existing functionality preserved

---

## Testing

- Feature manually tested
- Edge cases reviewed
- No known blocking issues

---

## Documentation

- Engineering Log updated
- Release documentation updated (if applicable)
- ADR created (if needed)
- Lessons Learned recorded (if applicable)

---

## Git

- Professional commit message
- Repository updated

---

## Engineering Review

Confirm:

- Architecture remains consistent
- Security considered
- Performance acceptable
- Code remains maintainable

---

## Mission Complete

Mission may only be marked complete after every checklist item has been satisfied.

---

# Engineering Philosophy

Done means:

Built

Tested

Documented

Committed

Ready for future engineers.
